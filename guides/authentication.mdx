## API keys

All endpoints require an API key in the `Authorization` header.

```http
Authorization: Bearer YOUR_API_KEY
```

## Example request

```bash
curl https://cc2.cx/v1/models \
  -H "Authorization: Bearer $AI_API_KEY"
```

## Security recommendations

- Store API keys in environment variables or a secrets manager.
- Call the API from your backend when possible.
- Rotate keys if they are leaked or no longer needed.
- Use separate keys for development, staging, and production.

<Warning>
  Do not ship API keys in frontend JavaScript, mini programs, mobile clients, or public Git repositories.
</Warning>
